A practical user management solution in Microsoft Access

Project Overview

This project demonstrates a practical SaaS solution built on Microsoft Access that prioritizes ease of use, effective and relevant dashboards; serving a direct and time-sensitive need for the client in managing their scattered databases and operations.

Problem Statement

Institutions--especially government--often have limitations on applications or solutions that they can build fast because of the formalities involved in asking a third party to build custom apps for a client--especially in the government sector. 

Methodology

  • Took time to learn basic operations from the target users (staff) in the department
  • Went over available documentation that outlines the processes that need to be automated, stored, and processed.
  • Analyzed the available workflows and potential areas for optimization
  • Developed a comprehensive and progressive solution that aims to provide valuable insights and help staff organize their operations.
  • Implemented an agile methodology where I integrate continuous feedback from the target users to make informed decisions about the features I build. 

Key implemented services

  • Incorporated dashboards to give overview stats and in-depth insights
  • Easy update of client information through workshop attendance, verification, and contextual information about them (institution, number of workshops, and any activities they've gone through).
  • Customized and ready-to-print reports that are dynamic to the requested data.
  • Future updates include user-access management and authentication and advanced reporting.

Recommendations

  • Implement a structured training program for customer service skills
  • Introduce ongoing technical training sessions for new technologies
  • Develop soft skills workshops focusing on communication and teamwork

Tools and Technologies Used

  • Development: Built-in design features and the VBA Scripting modules
  • Gathering resources: Microsoft Excel, Outlook, and related tools
  • Task management:  Notion

Challenges and Solutions

  • Challenge: Lack of familiarity with the VBA Scripting language
    • Solution: Used this project as a learning opportunity and utilized GenAI tools (like ChatGPT) to learn along the way and experiment with features and syntax
  • Challenge: Scattered user data across spreadsheets and emails
    • Solution: Used data analysis and data cleaning strategies to come up with a unified list and started feeding it to the system early on. I made changes as I go and when the data is available.

Project Impact

  • The staff, now that they can see a concrete and practical application that manages their data, gave them insights on how they can further improve their workflow and optimize it.
  • The created databases through this system are usable and scalable to bigger applications in the future (it uses SQL in the backend).

Lessons Learned

  • Focusing on the value of a product instead of how popular it is.
  • Value of employee feedback in shaping effective and value-added applications
