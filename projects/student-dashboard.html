Student Statistical Dashboard

Project Overview

This project utilizes custom APIs created in Google Sheets to extract required student body data into the Google Looker's database. The end result: a dynamic, self-updating dashboard ready to be used by college admins and decision makers.

Problem Statement

Student data was everywhere, but everything was in isolation. This meant that it's very hard to make decisions like future recruitment, student services, utilies and so on without having hard data showing growth or decline in areas of interest.

Methodology

  • A quick survery of the institution's digital infrastructure revealed that there is a lack of unity and centralization of data. This meant I need to establish a pipeline that gathers relevant data to be used on a dashboard.
  • The decision to go with Google Looker was simply because it was within the local ecosystem and it had robust data wrangling and processing.
  • I also created custom Apps Scrippt API to talk to data warehouses to bring in the data (SIS, spreadsheets, and other local data) and then use them to display content on the Google Looker dashboards. I also set up Cron jobs to control when the API is triggered to refresh the datasets.

Key implemented services

  • I created 4 pages to organize the data displayed into "themes": Enrollement Stats, Student profiles overview, APL Stats (Accreditation of Prior Learning), and Progress Stats to track students' journey from the first year till graduation.
